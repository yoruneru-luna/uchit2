Да, схема хорошая. Получается два режима набора:

```text
Набор с расписанием FSRS
→ участвует в «Повторить сегодня»
→ есть прогресс закрепления
→ после ответа пользователь ставит оценку
→ карточка получает следующую дату повторения

Обычный набор
→ только свободная тренировка
→ не участвует в «Повторить сегодня»
→ прогресс FSRS не считается
→ после прохождения просто поздравление
```

## Итоговая логика

### 1. Настройка набора: включать FSRS или нет

В форме создания/редактирования набора добавить переключатель:

```text
Добавлять набор в расписание повторений
```

Подпись:

```text
Если включено, карточки набора будут появляться в блоке «Повторить сегодня» и участвовать в прогрессе закрепления.
```

В БД у `study_sets`:

```php
fsrs_enabled boolean default true
fsrs_goal decimal(3, 2) nullable
```

`fsrs_goal` нужен только когда `fsrs_enabled = true`.

---

### 2. Цель изучения набора

Название лучше такое:

```text
Цель закрепления
```

Варианты:

```text
Лёгкая — карточки будут возвращаться реже
Стандартная — сбалансированный режим
Строгая — карточки будут возвращаться чаще
```

В коде:

```text
Лёгкая → 0.80
Стандартная → 0.90
Строгая → 0.95
```

Хранить лучше именно у набора, потому что разные наборы могут иметь разную важность.

Пример:

```text
Английский для себя → лёгкая
Экзамен → строгая
Основной учебный набор → стандартная
```

---

### 3. Если FSRS выключен

Для обычного набора:

```text
— набор не попадает в «Повторить сегодня»
— карточки не считаются в главном прогрессе
— оценки Again / Hard / Good / Easy не показываются
— в конце не показывается статистика «вспомнили / не вспомнили»
```

Конец режима:

```text
Набор пройден
Все карточки из набора просмотрены.
```

Для письменного и аудио режима можно оставить проверку ответа внутри карточки, но **не превращать результат в FSRS-оценку**. То есть «Правильно / Неправильно» можно показывать во время прохождения, но итоговую статистику FSRS не выводить.

---

### 4. Максимальный интервал

В интерфейсе лучше не показывать.

В системе оставить технически:

```text
Максимальный интервал: 365 дней
```

Пользователь не видит настройку, но система не даёт карточкам улетать на 10 лет вперёд.

Для прогресса «закреплено» не надо брать 365 дней. Лучше считать так:

```text
карточка закреплена, если scheduled_days >= 90
```

То есть:

```text
90+ дней → закреплена
меньше 90 дней → в процессе
due_at просрочен → забывается
нет progress → новая
```

Так «закреплено» будет понятным статусом, а максимальный интервал останется защитой от странных расчётов.

---

### 5. Глобальная настройка: сколько новых карточек вводить в изучение в день

В настройки пользователя добавить:

```text
Новые карточки в день
```

Подпись:

```text
Сколько ещё не изучавшихся карточек система будет добавлять в расписание повторений ежедневно.
```

Варианты:

```text
5 — спокойно
10 — стандартно
20 — интенсивно
30 — очень активно
```

Важно: это **не про создание карточек**, а про ввод уже созданных карточек в FSRS.

Пример:

```text
В наборе 100 карточек.
Лимит: 10 новых карточек в день.

Сегодня в расписание попадут 10 новых.
Завтра — ещё 10 новых + повторения старых.
```

---

## Что будет, если создано 5 наборов по 20 карточек

Не нужно брать по 10 из каждого набора.

Лимит должен быть **общий на аккаунт**.

Например:

```text
Новые карточки в день: 10
Есть 5 FSRS-наборов по 20 карточек

Система введёт всего 10 новых карточек за день, а не 50.
```

Порядок можно сделать простой:

```text
1. Сначала карточки, срок повторения которых уже наступил.
2. Потом новые карточки из FSRS-наборов.
3. Новых карточек добавляется не больше дневного лимита.
```

Позже можно добавить приоритет набора, но сейчас можно брать новые карточки по дате создания набора/карточки.

---

## Что делать, если набор был FSRS → обычный → снова FSRS

Лучше не удалять старый прогресс. Нужно **замораживать** расписание.

### Был FSRS, потом выключили

```text
— fsrs_enabled становится false
— записи card_review_progress не удаляются
— набор пропадает из «Повторить сегодня»
— прогресс FSRS скрывается
— ручная тренировка не меняет due_at, stability, difficulty
```

То есть история остаётся, но не используется.

### Через время снова включили FSRS

Лучший вариант: **продолжить с прошлого места**.

```text
— старые progress-записи снова начинают учитываться
— карточки с просроченным due_at попадают в повторение
— карточки без progress считаются новыми
— новые карточки вводятся по дневному лимиту
```

Так пользователь не теряет историю.

Но можно добавить предупреждение:

```text
У набора уже есть история повторений. После включения расписания карточки продолжат повторяться с прошлых дат.
```

Позже можно добавить кнопку:

```text
Сбросить прогресс набора
```

Но для первой версии лучше не усложнять.

---

## Что делать, если набор был обычным и стал FSRS

У карточек нет progress-записей, значит они считаются новыми.

После включения:

```text
— набор начинает участвовать в расписании
— система постепенно вводит карточки по дневному лимиту
— после первой оценки карточка получает due_at, stability, difficulty
```

---

## Что будет при ручном открытии набора

Нужны два сценария.

### FSRS включён

Если пользователь открыл набор вручную и запустил обучение:

```text
— оценки сохраняются в FSRS
— карточки получают новые даты повторения
— прогресс обновляется
```

То есть ручное прохождение тоже влияет на расписание.

### FSRS выключен

```text
— оценки не показываются
— FSRS не обновляется
— в конце просто «Набор пройден»
```

## Какие поля нужны

### `study_sets`

```php
fsrs_enabled boolean default true
fsrs_goal decimal(3, 2) nullable
```

### `users`

```php
daily_new_cards_limit unsignedInteger default 10
```

### Системная настройка в коде

Например в `config/fsrs.php`:

```php
return [
    'max_interval_days' => 365,
    'learned_threshold_days' => 90,
];
```

---

## Кратко: правильная архитектура

```text
Глобально:
— сколько новых карточек вводить в день

В наборе:
— участвует ли набор в FSRS
— цель закрепления набора

В системе:
— максимальный интервал 365 дней
— порог закрепления 90 дней
```

Дальше лучше начать с миграций: добавить `fsrs_enabled`, `fsrs_goal` в `study_sets` и `daily_new_cards_limit` в `users`.
